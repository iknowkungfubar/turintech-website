# Security Policy

## Supported Versions

This project is a static website. Security updates are applied to the `main` branch and deployed automatically via GitHub Pages.

| Version | Supported |
|---------|-----------|
| main    | ✅ Active |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in this project, please report it privately before disclosing it publicly.

**Do not** report security issues via public GitHub issues.

Instead, email: **contact@turintechsolutions.com**

You can expect:

1. **Acknowledgement** within 48 hours of your report
2. **Validation** and impact assessment within 5 business days
3. **A fix timeline** communicated once the issue is understood
4. **Credit** in the release notes (if you wish) once the issue is resolved

## Scope

This security policy covers:

- The Astro website source code
- CI/CD pipeline configuration (`.github/workflows/`)
- Build and deployment artifacts that run on GitHub Pages

## Out of Scope

- The live site's DNS or CDN configuration (handled via GitHub Pages)
- Third-party services linked from the site (e.g., contact webhook endpoints)

## Safe Harbor

We will not pursue legal action against researchers who:

- Follow this disclosure policy
- Do not access or modify user data without permission
- Do not interrupt or degrade the service
- Act in good faith to improve security
